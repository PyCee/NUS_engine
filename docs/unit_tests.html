Unit Tests are simple programs designed to test specific functions of the NUS library. These programs may rely on code originally tested in other unit tests, or may be independant of other unit tests.
save_file
obtains information from a save file, prints the information out, and then saves the file with an incremented number from the file. A test for basic save file input/output.
source
3d_math
Tests various 3d math functions for correctness.
source
gravity_simulation
Uses 3d_math to determine the effect of gravity on a resting object and prints the result. Tests force and momentum effects on a mass.
source
basic_window
Opens a window and detects input. The window can be closed by pressing the esc key or clicking the close window button that is usually in the top-left or top-right of the displayed window (depending on your operating system). Tests window creation and input detection.
source
gpu_detection
Utilizes vulkan to detect, and then print information regarding the client's graphical processing unit. This program will open a window to get some information and automatically close said window once the information has been obtained. Tests vulkan instance creation, presentation surface creation, and obtaining gpu information.
source
vulkan_clear
Similar to basic_window, this program opens a window and can be closed by the esc key or the close button. Vulkan is utilized to make the computer's gpu clear the window to a solid color. Tests basic vulkan command submiting and surface presenting.
source
hello_triangle
A program that builds upon vulkan_clear. It uses shaders to draw a solid colored triangle and display it on a window. Tests shader utilization without uniform memory variables. Also tests loading a NUSM model into application memory.
source
uniform_shader
Building off of hello_triangle, this program displays and moves a triangle around the screen based on keyboard input (the wasd and arrow keys). Tests basic vulkan uniform variables in shaders and description sets.
