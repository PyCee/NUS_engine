Unit Tests are simple programs designed to test specific functions of the NUS library. These programs may rely on code originally tested in other unit tests, or may be independant of other unit tests.
save_file
obtains information from a save file, prints the information out, and then saves the file with an incremented number from the file. A test for basic save file input/output
source
3d_math_vector
a program that computes various vector operations
source
3d_math_matrix
a program that computes matrix operations
source
3d_math_axes
a program that computes 3d local axis operations
source
3d_math_reference_vector
a program that computes various refernce vector operations
source
gravity_simulation
uses 3d_math to determine the efect of gravity on a resting object and prints the result. Tests force and momentum effects on a mass
source
basic_window
opens a window and detects input. window can be closed by pressing the esc key or clicking the close window button that is usually in the top-left or top-right (dependant on your operating system). Tests window creation and input detection
source
gpu_detection
utilizes vulkan to detect, and then print, information regarding the clients graphical processing unit. This program will open a window to get some information and automatically close said window once the information has been obtained. Tests vulkan instnace creation, presentation surface creation, and obtaining gpu information
source
vulkan_clear
similar to basic_window, this program opens a window and can be closed by the esc key or the close button. vulkan is utilized to make a gpu clear the window to a solid color. Tests basic vulkan command submiting and surface presenting
source
hello_triangle
a program that build upon vulkan_clear and uses shaders to draw a solid colored triangle, then display it on a window. Tests shader utilization without uniform memory
